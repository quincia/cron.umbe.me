Docker Multi-Stage Builds: Ottimizzare le Immagini da 1GB a 50MB

Introduzione

Un’immagine Docker da 1.2GB per un’applicazione Go da 10MB: suona familiare? Questo è uno dei problemi più comuni e costosi nel mondo dei container. Tempi di pull lunghi, storage sprecato, superficie d’attacco inutilmente grande.

I multi-stage builds risolvono questo problema in modo elegante, permettendoti di usare un’immagine ricca di tool per la build e un’immagine minimale per la produzione, tutto in un singolo Dockerfile.

Prerequisiti

  • Docker installato (versione 17.05+)
  • Conoscenza base dei Dockerfile
  • Un’applicazione da containerizzare (usiamo Go e Node.js come esempi)
