Event-Driven Architecture in Pimcore: Building Asynchronous Workflow Systems with Symfony Messenger

Every Pimcore developer eventually hits the same wall. You’re importing 50,000 products from an ERP, generating thumbnails for 10,000 assets, or synchronizing data across five e-commerce channels—and the admin interface becomes unresponsive. Users stare at spinning loaders. Timeouts kill long-running operations. The system feels brittle.

The solution isn’t faster hardware. It’s architectural: decouple the triggering of operations from their execution. This is where event-driven architecture shines, and Pimcore’s native integration with Symfony Messenger provides the foundation to build it properly.

This article walks through building production-grade asynchronous workflows in Pimcore. We’ll cover multi-transport configuration, custom event subscribers, resilience patterns, and orchestration strategies for complex PIM/DAM operations. By the end, you’ll have a blueprint for handling massive data operations without blocking users or risking data inconsistency.

Prerequisites

To follow along, you’ll need:

  • Pimcore 11.x with Symfony 6.4+
  • Working knowledge of Symfony services and dependency injection
  • A message broker installed (Redis or RabbitMQ—we’ll cover both)
  • Supervisor or systemd for worker process management
  • Basic understanding of event-driven concepts

Your development environment should have:

  • PHP 8.2+ with the appropriate extensions (redis, amqp)
  • Composer for dependency management
  • Access to Pimcore’s console commands

            async_fallback:
                dsn: 'doctrine://default?table_name=messenger_messages'
                retry_strategy:
                    max_retries: 10
                    delay: 60000
                    
            # Dead letter queue for manual intervention
            failed:
                dsn: 'doctrine://default?table_name=messenger_failed_messages'

        # Route messages to appropriate transports
        routing:
            # Asset operations go to fast transport
            'App\Messenger\Message\Asset\AssetPostProcessMessage': async_fast
            
            # Import operations need reliability
            'App\Messenger\Message\Import\ProcessImportBatchMessage': async_reliable
            
            # Channel sync operations
            'App\Messenger\Message\Sync\ChannelSyncMessage': async_reliable
            
            # Pimcore's internal async messages
            'Pimcore\Messenger\AssetUpdateTasksMessage': async_fast
            'Pimcore\Messenger\SearchBackendMessage': async_fast

💡 The transport selection strategy matters: Redis excels at high-throughput ephemeral tasks (regenerating a thumbnail can always be retried). RabbitMQ provides delivery guarantees essential for operations with external side effects (sending data to an ERP that doesn’t handle duplicates gracefully).

<?php

declare(strict_types=1);

namespace App\EventSubscriber;

use App\Messenger\Message\Asset\AssetPostProcessMessage;
use Pimcore\Event\AssetEvents;
use Pimcore\Event\Model\AssetEvent;
use Pimcore\Model\Asset;
use Symfony\Component\EventDispatcher\EventSubscriberInterface;
use Symfony\Component\Messenger\MessageBusInterface;
use Symfony\Component\Messenger\Stamp\DelayStamp;
use Symfony\Component\Uid\Uuid;
use Psr\Log\LoggerInterface;

final class AssetWorkflowSubscriber implements EventSubscriberInterface
{
    // Skip processing for these asset types (processed elsewhere or not needed)
    private const EXCLUDED_TYPES = ['folder'];
    
    // Minimum file size (bytes) to warrant async processing
    private const ASYNC_THRESHOLD_BYTES = 102400; // 100KB

    public function __construct(
        private readonly MessageBusInterface $messageBus,
        private readonly LoggerInterface $logger
    ) {}

    public static function getSubscribedEvents(): array
    {
        return [
            AssetEvents::POST_ADD => ['onAssetPostAdd', -100],    // Low priority: run after Pimcore's handlers
            AssetEvents::POST_UPDATE => ['onAssetPostUpdate', -100],
        ];
    }

    public function onAssetPostAdd(AssetEvent $event): void
    {
        $asset = $event->getAsset();
        
        if (!$this->shouldProcessAsync($asset)) {
            return;
        }

        $correlationId = Uuid::v4()->toRfc4122();
        
        $this->logger->info('Dispatching asset post-process message', [
            'assetId' => $asset->getId(),
            'assetType' => $asset->getType(),
            'correlationId' => $correlationId,
            'fileSize' => $asset->getFileSize(),
        ]);

        $message = new AssetPostProcessMessage(
            assetId: $asset->getId(),
            assetType: $asset->getType(),
            operations: $this->determineOperations($asset),
            priority: $this->calculatePriority($asset),
            correlationId: $correlationId
        );

        // Delay processing slightly to ensure database transaction is committed
        $this->messageBus->dispatch($message, [
            new DelayStamp(500) // 500ms delay
        ]);
    }

    public function onAssetPostUpdate(AssetEvent $event): void
    {
        $asset = $event->getAsset();
        
        // Only reprocess if binary data changed
        if (!$asset->getDataChanged()) {
            return;
        }
        
        if ($this->shouldProcessAsync($asset)) {
            $message = new AssetPostProcessMessage(
                assetId: $asset->getId(),
                assetType: $asset->getType(),
                operations: ['thumbnails'], // Only regenerate thumbnails on update
                correlationId: Uuid::v4()->toRfc4122()
            );

            $this->messageBus->dispatch($message, [
                new DelayStamp(500)
            ]);
        }
    }

    private function shouldProcessAsync(Asset $asset): bool
    {
        // Exclude certain types
        if (in_array($asset->getType(), self::EXCLUDED_TYPES, true)) {
            return false;
        }

        // Only async for larger files
        $fileSize = $asset->getFileSize();
        if ($fileSize !== null && $fileSize < self::ASYNC_THRESHOLD_BYTES) {
            return false;
        }

        return true;
    }

    private function determineOperations(Asset $asset): array
    {
        $operations = [];

        if ($asset instanceof Asset\Image) {
            $operations[] = 'thumbnails';
            $operations[] = 'metadata';
            $operations[] = 'color_extraction';
        } elseif ($asset instanceof Asset\Video) {
            $operations[] = 'thumbnails';
            $operations[] = 'metadata';
            $operations[] = 'transcode';
        } elseif ($asset instanceof Asset\Document) {
            $operations[] = 'preview';
            $operations[] = 'text_extraction';
        }

        return $operations;
    }

    private function calculatePriority(Asset $asset): int
    {
        // Higher priority for smaller files (faster to process)
        $fileSize = $asset->getFileSize() ?? 0;
        
        if ($fileSize < 1048576) { // < 1MB
            return 10;
        } elseif ($fileSize < 10485760) { // < 10MB
            return 5;
        }
        
        return 0;
    }
}

⚠️ The 500ms delay stamp is critical. Without it, your message handler might try to load an asset before the database transaction that created it has committed. This causes intermittent “Asset not found” errors that are maddening to debug.

<?php

declare(strict_types=1);

namespace App\Messenger\Handler\Asset;

use App\Messenger\Message\Asset\AssetPostProcessMessage;
use Pimcore\Model\Asset;
use Pimcore\Model\Asset\Image\Thumbnail;
use Symfony\Component\Messenger\Attribute\AsMessageHandler;
use Symfony\Component\Messenger\Exception\RecoverableMessageHandlingException;
use Symfony\Component\Messenger\Exception\UnrecoverableMessageHandlingException;
use Psr\Log\LoggerInterface;
use Doctrine\DBAL\Connection;

#[AsMessageHandler]
final class AssetPostProcessHandler
{
    public function __construct(
        private readonly LoggerInterface $logger,
        private readonly Connection $connection,
        private readonly array $thumbnailConfigs = []
    ) {}

    public function __invoke(AssetPostProcessMessage $message): void
    {
        $this->logger->info('Processing asset post-process message', [
            'assetId' => $message->getAssetId(),
            'operations' => $message->getOperations(),
            'correlationId' => $message->getCorrelationId(),
        ]);

        // Clear Pimcore's runtime cache to get fresh data
        \Pimcore\Cache\RuntimeCache::clear();

        $asset = Asset::getById($message->getAssetId());
        
        if ($asset === null) {
            // Check if asset was deleted after message was dispatched
            if ($this->assetWasDeleted($message->getAssetId())) {
                $this->logger->info('Asset was deleted, skipping processing', [
                    'assetId' => $message->getAssetId(),
                ]);
                return; // Not an error, just skip
            }
            
            // Asset might not be committed yet, retry
            throw new RecoverableMessageHandlingException(
                sprintf('Asset %d not found, will retry', $message->getAssetId())
            );
        }

        try {
            foreach ($message->getOperations() as $operation) {
                $this->executeOperation($asset, $operation);
            }
            
            $this->logger->info('Asset processing completed', [
                'assetId' => $message->getAssetId(),
                'correlationId' => $message->getCorrelationId(),
            ]);
        } catch (\ImagickException $e) {
            // Image processing errors are usually unrecoverable (corrupt file, unsupported format)
            throw new UnrecoverableMessageHandlingException(
                sprintf('Image processing failed for asset %d: %s', $asset->getId(), $e->getMessage()),
                previous: $e
            );
        } catch (\RuntimeException $e) {
            // Generic runtime errors might be transient
            throw new RecoverableMessageHandlingException(
                sprintf('Processing failed for asset %d, will retry: %s', $asset->getId(), $e->getMessage()),
                previous: $e
            );
        }
    }

    private function executeOperation(Asset $asset, string $operation): void
    {
        match ($operation) {
            'thumbnails' => $this->generateThumbnails($asset),
            'metadata' => $this->extractMetadata($asset),
            'color_extraction' => $this->extractColors($asset),
            'preview' => $this->generatePreview($asset),
            'text_extraction' => $this->extractText($asset),
            'transcode' => $this->transcodeVideo($asset),
            default => $this->logger->warning('Unknown operation', ['operation' => $operation]),
        };
    }

    private function generateThumbnails(Asset $asset): void
    {
        if (!$asset instanceof Asset\Image) {
            return;
        }

        $configs = $this->thumbnailConfigs ?: $this->getDefaultThumbnailConfigs();
        
        foreach ($configs as $configName) {
            $config = Thumbnail\Config::getByName($configName);
            if ($config !== null) {
                // Generate thumbnail synchronously in worker
                $asset->getThumbnail($configName, false);
                
                $this->logger->debug('Generated thumbnail', [
                    'assetId' => $asset->getId(),
                    'config' => $configName,
                ]);
            }
        }
    }

    private function extractMetadata(Asset $asset): void
    {
        if (!$asset instanceof Asset\Image) {
            return;
        }

        // Force metadata extraction
        $asset->getMetadata();
        $asset->getCustomSettings();
        
        // Extract EXIF data if available
        $dimensions = $asset->getDimensions();
        if ($dimensions) {
            $this->logger->debug('Extracted image dimensions', [
                'assetId' => $asset->getId(),
                'width' => $dimensions['width'] ?? null,
                'height' => $dimensions['height'] ?? null,
            ]);
        }
    }

    private function extractColors(Asset $asset): void
    {
        if (!$asset instanceof Asset\Image) {
            return;
        }

        // Implement color palette extraction using Imagick
        // Store results in custom metadata
        try {
            $imagePath = $asset->getLocalFile();
            if ($imagePath && file_exists($imagePath)) {
                $colors = $this->extractDominantColors($imagePath);
                
                $customSettings = $asset->getCustomSettings() ?? [];
                $customSettings['dominantColors'] = $colors;
                $asset->setCustomSettings($customSettings);
                $asset->save();
            }
        } catch (\Exception $e) {
            $this->logger->warning('Color extraction failed', [
                'assetId' => $asset->getId(),
                'error' => $e->getMessage(),
            ]);
        }
    }

    private function extractDominantColors(string $imagePath): array
    {
        $imagick = new \Imagick($imagePath);
        $imagick->quantizeImage(5, \Imagick::COLORSPACE_RGB, 0, false, false);
        $imagick->uniqueImageColors();
        
        $colors = [];
        $width = $imagick->getImageWidth();
        for ($i = 0; $i < $width && $i < 5; $i++) {
            $pixel = $imagick->getImagePixelColor($i, 0);
            $colors[] = $pixel->getColor();
        }
        
        $imagick->destroy();
        
        return $colors;
    }

    private function generatePreview(Asset $asset): void
    {
        if (!$asset instanceof Asset\Document) {
            return;
        }
        
        // Generate document preview/thumbnail
        $asset->getThumbnail(Asset\Document\ImageThumbnail::getPreviewConfig());
    }

    private function extractText(Asset $asset): void
    {
        if (!$asset instanceof Asset\Document) {
            return;
        }
        
        // Text extraction would typically use a library like Smalot\PdfParser
        // or shell out to pdftotext
        $this->logger->debug('Text extraction placeholder', ['assetId' => $asset->getId()]);
    }

    private function transcodeVideo(Asset $asset): void
    {
        if (!$asset instanceof Asset\Video) {
            return;
        }
        
        // Video transcoding would typically use FFmpeg
        $this->logger->debug('Video transcode placeholder', ['assetId' => $asset->getId()]);
    }

    private function assetWasDeleted(int $assetId): bool
    {
        $result = $this->connection->fetchOne(
            'SELECT id FROM assets WHERE id = ?',
            [$assetId]
        );
        
        return $result === false;
    }

    {
